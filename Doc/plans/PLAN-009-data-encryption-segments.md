**预估时间：** 1 天

- **文件**：
  - 创建：`lib/migration/enable_pgcrypto.py`
- **测试策略**：测试后行

- **文件**：

- **测试策略**：TDD
  - 理由：
    - 核心业务逻辑（加密模块）

- **验收标准**：
  - 支持应用层加密/解密

  - 支持 pgcrypto SQL 生成
  - 空值处理正确
  - 密钥缺失时抛出异常

- **复杂度**：高

**类型：** 串行
**前置条件：** 任务组 1 完成

- **文件**：

- **验收标准**：

  - phone_hmac 和 email_hmac 字段已创建索引
- **依赖**：任务组 1
- **复杂度**：中

#### SUB-TASK-005：修改用户模型

- **描述**：在 User 模型中添加加密字段，实现属性访问器自动加解密
- **文件**：
  - 修改：`lib/models/user.py`
- **测试策略**：TDD
  - 理由：
    - 核心业务逻辑（用户模型）
    - 数据处理和转换（属性访问器）
- **验收标准**：
  - 模型包含 phone_encrypted、email_encrypted、phone_hmac、email_hmac 字段
  - phone 和 email 属性访问器支持自动加解密
  - 空值处理正确
- **依赖**：SUB-TASK-004
- **复杂度**：中

#### SUB-TASK-006：修改用户服务（加密逻辑）

- **描述**：修改用户服务，在创建、查询、更新用户时自动处理加密字段
- **文件**：
  - 修改：`lib/services/user_service.py`
- **测试策略**：TDD
  - 理由：
    - 核心业务逻辑（服务层）
    - API 契约（CRUD 操作）
    - 安全相关（敏感数据处理）
- **验收标准**：
