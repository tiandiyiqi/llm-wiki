## [未发布] - 2026-06-25

### 改进（Improved）— PLAN-M-012 侧边栏治理与 UI 组件化

- **导航统一**：桌面侧边栏与移动端抽屉改由单一配置（`views/utils/nav-config.js`）驱动，彻底消除「桌面 9 项 vs 移动 4 项」的入口不一致。
- **重型模块入口补齐**：权限、知识库、用户、审批、审计模块现可从侧边栏「系统管理」分组按角色进入。
- **侧边栏二级分组**：导航项按「内容操作 / 系统管理」分组呈现，结构更清晰。
- **共享 UI 组件库**：新增 `views/utils/ui-components.js`，提供统一的 HTML 转义、徽章、加载指示器、弹窗、表格、选项卡组件（默认转义，自带主题适配）。
- **代码去重**：13 个视图模块的本地 `escapeHtml` 实现统一为单一共享实现。
- **看板主题化**：数据看板（dashboard）统计卡与图表全面接入 5 套主题，主题切换时图表自动重绘。

### 修复（Fixed）

- **「通知」入口语义错配**：旧侧边栏「通知」（铃铛图标）实际跳转到 Webhook 配置页。现拆分为「通知」（站内消息中心）与「集成 / Webhook」（出站推送配置）两个独立入口。
- **看板图表重复渲染报错**：刷新看板时未先销毁旧图表实例导致的 "Canvas is already in use" 错误已修复。
- **侧边栏权限可见性失效**：旧权限控制脚本因缺少初始隐藏类而从未生效，现改由角色白名单 + Alpine `x-show` 真正按角色控制可见性。

### 变更（Changed）

- **文件上传 API 统一**：文件上传改用新增的 `WikiAPI.postForm`（正确处理 multipart 边界 + 未登录跳转），与全站 API 调用方式统一。
- **旧管理页迁移提示**：14 个旧 `admin/*.html` 静态页保留并新增「前往新版」提示横幅，活跃 SPA 代码中的旧页链接引用已清零（保护外部书签不失效）。
