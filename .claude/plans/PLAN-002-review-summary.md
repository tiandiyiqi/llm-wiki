## 🚨 P0 - 立即修复（阻塞发布）

### 安全问题（CRITICAL）

**1. 不安全的密码哈希算法**
- **文件**: `lib/auth.py:54-64`
- **问题**: SHA-256 + 固定盐，易破解
- **修复**: 使用 bcrypt 或 argon2

**2. SQL 拼接导致注入风险**
- **文件**: `lib/core/db_storage.py:98-112`, `lib/auth/rls_manager.py:75-96`
- **问题**: 字符串拼接构建 SQL
- **修复**: 完全参数化查询 + 白名单验证

**3. 缺少认证的 API 端点**
- **文件**: `lib/api_server.py:52-227`
- **问题**: 所有端点公开访问
- **修复**: 实现认证中间件

**4. RLS 策略 SQL 注入**
- **文件**: `lib/auth/rls_manager.py:75-96`
- **问题**: 直接拼接 kb_id 到 SQL
- **修复**: 严格类型检查 + quote_ident

### 代码质量问题（CRITICAL）

**5. 硬编码的密码盐值**
- **文件**: `lib/auth.py:63`
- **问题**: 固定盐泄露后全系统失效
- **修复**: 使用 bcrypt 自动生成盐

**6. SQL 拼接风险**
- **文件**: `lib/auth/rls_manager.py:46`
- **问题**: 动态拼接表名
- **修复**: 白名单验证

**7. 权限检查缺失**
- **文件**: `lib/api/kb_management.py:106-108`
- **问题**: 使用错误的 kb_id=0 检查权限
- **修复**: 实现系统级权限检查
